# LaborAny v0.4.0

发布日期：2026-03-12

---

## 版本重点

`v0.4.0` 重点修复并打通了记忆系统的关键链路，目标是让“用户说过一次，系统后续真的记得住”。

这次版本主要覆盖四个方向：

1. 称呼记忆
2. 默认回复偏好
3. 长期记忆写入与审计
4. 真实 UI 回归验证

---

## 主要更新

### 1. 称呼记忆升级

- 支持自动提取用户希望被如何称呼。
- 支持手动设置与自动学习并存。
- “你现在叫我什么？”这类元问题会直接返回当前称呼，不再误当成设置指令。
- 当用户指定名字后，后续对话不再统一回退到“老板”。

### 2. 回复偏好升级

- 支持自动学习默认回复语言（中文 / 英文）。
- 支持自动学习默认回复风格（简洁 / 详细）。
- 新增“结论优先”结构化沟通偏好归一：
  - 先给结论再给步骤
  - 先说结论再展开细节
  - 结论先行

这类偏好不再错误折叠成“偏好详细回复”。

### 3. 长期记忆链路修复

- 修复了长期记忆候选无法顺利进入全局/技能长期记忆的问题。
- 修复了同一会话多轮证据不会累计的问题。
- 修复了模型抽取超时但 stdout 已有合法 JSON 时仍被错误丢弃的问题。
- 自动写入长期记忆后会清理对应候选，避免悬挂候选残留。
- 沟通类稳定偏好在满足条件后可以同步写入技能级和全局长期记忆。

### 4. 记忆页面可见性升级

`/memory` 页面现在可以直接看到：

- 长期记忆统计
- 候选数量
- 最近审计日志
- 候选确认 / 忽略
- 历史审计回填

### 5. 验证体系升级

新增两类隔离回归：

- `npm run verify:memory-fastpaths`
- `npm run verify:memory-ui-real`

覆盖场景包括：

- 称呼更新
- 默认回复偏好更新
- “结论先行”长期记忆写入
- “称呼 + 长期偏好”组合链路

---

## 对用户的直接影响

- 你说“以后叫我 Nathan”，系统后续会按名字称呼。
- 你说“以后先给结论再展开”，系统后续会稳定按这个结构回复。
- 连续两次表达同一偏好后，系统会把它沉淀进长期记忆，而不是只停留在临时上下文。
- 记忆是否写入、为什么没写入，现在可以在 UI 中直接查看。

---

## 升级建议

如果你从旧版本升级到 `v0.4.0`：

1. 建议在 `/memory` 页面检查一次现有画像和长期记忆。
2. 如需补齐旧数据，可使用“回填历史审计”。
3. 如在本地开发，建议跑一次：

```bash
npm run verify:memory-fastpaths
node scripts/verify-memory-ui-real.js
```
