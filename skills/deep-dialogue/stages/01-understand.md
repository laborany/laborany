# 阶段1：理解与准备

## 目标

深度理解用户想讨论的话题，建立知识基础，以一个有质量的回应开启对话。

## 前置条件

- 用户发起了对话请求（可能是一个问题、一个想法、一个困惑）

---

## 步骤1：话题识别

分析用户的输入，识别：
- 核心话题是什么
- 话题类型：技术探讨 / 创业想法 / 哲学思辨 / 认知提升 / 健康规划 / 自由探索
- 用户的立场或倾向（如果有的话）
- 用户期望的对话深度

## 步骤2：快速知识准备

根据话题，使用搜索工具建立知识基础：

```
使用 mcp__laborany_web__search 搜索：
- "{话题} 最新进展 2025 2026"（中文）
- "{topic} latest research findings"（英文）
```

如果话题涉及具体事实或数据，额外搜索验证：
```
使用 mcp__laborany_web__search 搜索具体事实
使用 mcp__laborany_web__read_page 阅读 1-2 篇最相关的文章
```

注意：
- 搜索是为了建立知识基础，不是为了写报告
- 不需要告诉用户你在搜索（除非搜索结果直接相关）
- 如果话题是纯思辨性的（如哲学问题），可以跳过搜索

## 步骤3：形成初步观点

基于你的知识和搜索结果：
- 形成你自己对这个话题的独立观点
- 识别用户观点中可能的盲点或值得深挖的点
- 准备 1-2 个有深度的追问

## 步骤4：开启对话

你的第一轮回复应该包含：
1. 对用户话题的深度回应（不是简单复述，而是你的分析和观点）
2. 如果发现事实性问题，指出并提供正确信息
3. 一个有深度的追问，引导对话深入

避免：
- "这是个好问题"之类的空话
- 长篇大论的背景介绍
- 列出一堆要点但没有深度
- 急于给出"答案"或"建议"
