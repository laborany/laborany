## 核心原则

### 1. 简洁至上

> "Perfection is achieved not when there is nothing more to add, but when there is nothing left to take away." — Antoine de Saint-Exupéry

- 每个元素都必须有存在的理由
- 删除所有装饰性元素
- 信息密度要适中，留白是设计的一部分

### 2. 一致性

- 同类元素使用相同样式
- 间距、字体、颜色保持统一
- 箭头方向和样式一致

### 3. 层次分明

- 主要信息突出显示
- 次要信息弱化处理
- 使用大小、颜色、粗细区分层级

### 4. 可读性优先

- 字体大小不小于 12px
- 对比度足够（黑白打印也能看清）
- 避免过长的文字标签

## 流程图规范

### 标准符号

| 符号 | 形状 | 用途 |
|------|------|------|
| 开始/结束 | 椭圆/圆角矩形 | 流程的起点和终点 |
| 处理 | 矩形 | 操作、计算、处理步骤 |
| 判断 | 菱形 | 条件分支、决策点 |
| 数据 | 平行四边形 | 输入/输出数据 |
| 子流程 | 双边矩形 | 预定义的子过程 |

### 布局原则

```
1. 流向一致
   - 从上到下（推荐）
   - 从左到右（备选）
   - 避免混合方向

2. 对齐规则
   - 同级节点水平/垂直对齐
   - 连线尽量走直线
   - 避免交叉

3. 间距规范
   - 节点间距: 40-60px
   - 分支间距: 80-120px
   - 边距: 20-40px
```

### 标签规范

```
节点标签:
- 使用动词短语: "计算结果"、"验证输入"
- 简洁明了，不超过 10 个字
- 避免技术术语堆砌

判断标签:
- 使用疑问句: "是否有效？"
- 分支标注: "是/否"、"Y/N"、"True/False"

连线标签:
- 只在必要时添加
- 放置在连线中点附近
```

## 架构图规范

### 层级结构

```
┌─────────────────────────────────────┐
│           表现层 (Presentation)      │
├─────────────────────────────────────┤
│           业务层 (Business)          │
├─────────────────────────────────────┤
│           数据层 (Data)              │
├─────────────────────────────────────┤
│           基础设施 (Infrastructure)   │
└─────────────────────────────────────┘
```

### 模块表示

```
┌──────────────┐
│   模块名称    │
├──────────────┤
│  • 功能1     │
│  • 功能2     │
│  • 功能3     │
└──────────────┘
```

### 连接关系

| 线型 | 含义 |
|------|------|
| 实线箭头 | 依赖、调用 |
| 虚线箭头 | 可选依赖、异步调用 |
| 双向箭头 | 双向通信 |
| 无箭头线 | 关联关系 |

### 分组规范

- 相关模块用边框分组
- 分组添加标题说明
- 不同分组使用不同底色区分

## 时序图规范

### 基本元素

```
参与者 (Actor)
│
│  ─────────────────────>  同步消息
│
│  - - - - - - - - - - ->  异步消息
│
│  <─ ─ ─ ─ ─ ─ ─ ─ ─ ─   返回消息
│
├──┐
│  │  自调用
│<─┘
│
█████  激活条 (Activation)
```

### 布局原则

```
1. 参与者排列
   - 按交互顺序从左到右排列
   - 主要参与者放在左侧
   - 间距均匀

2. 消息排列
   - 从上到下按时间顺序
   - 同时发生的消息水平对齐
   - 避免消息线交叉

3. 标注规范
   - 消息名称放在箭头上方
   - 参数放在括号内
   - 返回值用虚线箭头
```

## 配色指南

### 学术论文配色

推荐使用黑白灰配色，确保打印效果：

```
主色: #1f2937 (深灰)
辅色: #6b7280 (中灰)
背景: #ffffff (白色)
边框: #374151 (灰色)
```

### 彩色配色原则

如果使用彩色，遵循以下原则：

```
1. 色彩数量
   - 主色 1 种
   - 辅色 2-3 种
   - 总计不超过 5 种颜色

2. 色彩含义
   - 绿色: 开始、成功、正常流程
   - 红色: 结束、错误、异常流程
   - 黄色: 警告、判断、条件
   - 蓝色: 处理、操作、中性

3. 对比度
   - 文字与背景对比度 > 4.5:1
   - 相邻元素有明显区分
```

### 色盲友好配色

避免仅用红绿区分，添加形状或图案辅助：

```
推荐组合:
- 蓝色 + 橙色
- 蓝色 + 黄色
- 紫色 + 绿色
```

## 排版规范

### 字体选择

```
中文: 思源黑体、微软雅黑、苹方
英文: Arial、Helvetica、Roboto
代码: Consolas、Monaco、Source Code Pro
```

### 字号规范

```
图表标题: 16-20px, 粗体
节点文字: 12-14px, 常规
标签文字: 10-12px, 常规
注释文字: 10px, 斜体
```

### 对齐规范

```
- 文字居中对齐
- 节点网格对齐
- 连线端点对齐节点中心
```

## 常见错误

### ❌ 错误示例

1. **信息过载**
   - 单图超过 15 个节点
   - 文字过长，挤满节点
   - 连线交叉混乱

2. **样式不一致**
   - 同类节点大小不同
   - 字体混用
   - 颜色随意

3. **布局混乱**
   - 流向不一致
   - 间距不均匀
   - 没有对齐

4. **过度设计**
   - 添加不必要的装饰
   - 使用 3D 效果
   - 渐变色滥用

### ✅ 正确做法

1. **控制复杂度**
   - 单图 5-10 个核心节点
   - 复杂流程拆分为多图
   - 使用子流程封装细节

2. **保持一致**
   - 建立样式规范并遵守
   - 使用模板
   - 复用组件

3. **注重细节**
   - 检查对齐
   - 统一间距
   - 校对文字
