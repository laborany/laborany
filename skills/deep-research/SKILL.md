# Deep Research

## Overview

将用户的研究请求转化为系统性的深度研究报告。核心流程：

1. 分析请求 → 识别研究类型、关键实体、复杂度
2. 生成研究提示词 → 选择框架、拆解子问题、定义信息源
3. 用户确认提示词 → 支持快速调整
4. 执行多源搜索 → 广度搜索 + 平台定向搜索 + 深度阅读
5. 整合信息 → 交叉验证 → 产出带引用链接的 Markdown 报告

## Workflow

按以下阶段顺序执行，每个阶段的详细指令见 `stages/` 目录下对应文件。

### 阶段1：分析请求

详见 `stages/01-analyze-request.md`

用户提出研究需求后：
- 识别核心研究主题，提取关键词
- 判断研究类型（事实收集 / 趋势分析 / 观点整理 / 深度研究 / 探索了解）
- 识别关键实体（人物、事件、组织、时间、地点、主题）
- 评估研究复杂度（简单 / 中等 / 复杂）
- 向用户展示分析摘要

### 阶段2：生成研究提示词

详见 `stages/02-generate-prompt.md`

基于阶段1的分析结果，生成包含4个优化维度的完整提示词：
- 维度1：研究框架/结构（根据研究类型选择，详见 `templates/research-framework.md`）
- 维度2：具体子问题拆解（3-5个核心子问题）
- 维度3：信息来源指引（定制信息源类型和质量要求）
- 维度4：输出格式要求（详见 `templates/output-format.md`）

组装完整提示词后展示给用户确认。

### 阶段3：确认与调整

详见 `stages/03-refine-adjust.md`

将生成的提示词展示给用户确认：
- 用户回复"可以"、"好"、"没问题"等 → 进入阶段4
- 用户提出修改意见 → 调整后再次确认
- 支持调整详细程度（快速版 / 标准版 / 完整版）
- 支持调整维度、输出格式等
- 最多迭代2轮调整

### 阶段4：执行研究

详见 `stages/04-execute-research.md`

这是核心执行阶段：

1. 确认报告保存路径（建议 `./docs/{主题关键词}-{YYYY-MM-DD}.md`）
2. 制定多源搜索计划（覆盖中英文平台）
3. 执行搜索：
   - 第一轮：广度搜索 + 平台定向搜索（`mcp__web-search-prime__webSearchPrime`）
   - 第二轮：深度阅读 5-8 篇最相关文章（`mcp__web-reader__webReader`）
   - 第三轮：补充搜索（按需，最多3次）
4. 信息整理与交叉验证
5. 合成研究报告（含 YAML frontmatter、行内/脚注引用、来源汇总、研究局限性）
6. 保存并展示摘要
7. 询问后续操作

## 搜索平台覆盖

### 中文平台（中文主题必搜）
| 平台 | 搜索方式 | 适合内容 |
|------|---------|---------|
| 知乎 | `site:zhihu.com {关键词}` | 深度讨论、专业观点 |
| CSDN | `site:csdn.net {关键词}` | 技术文章 |
| 掘金 | `site:juejin.cn {关键词}` | 前端/开发技术 |
| 36氪 | `site:36kr.com {关键词}` | 商业分析、创投 |
| 少数派 | `site:sspai.com {关键词}` | 产品体验、效率工具 |

### 英文平台（英文主题必搜）
| 平台 | 搜索方式 | 适合内容 |
|------|---------|---------|
| Reddit | `site:reddit.com {keywords}` | 社区讨论、用户反馈 |
| Twitter/X | `site:x.com OR site:twitter.com {keywords}` | 实时动态 |
| GitHub | `site:github.com {keywords}` | 开源项目、技术实现 |
| Hacker News | `site:news.ycombinator.com {keywords}` | 技术社区讨论 |
| Medium | `site:medium.com {keywords}` | 深度文章 |

### 通用来源（所有研究必搜）
- 官方网站/文档、权威媒体、学术/报告

## 引用规则

报告中所有关键信息必须附带可点击的引用链接：
- 行内引用：关键论断后直接附带来源链接
- 脚注引用：密集引用段落使用 `[^1]` 格式
- 每个主要段落至少1个引用链接
- 数据/统计、直接引用、观点归属必须有引用
- 来源按平台分类汇总

## Key Principles

1. 好的研究始于好的问题，花时间优化 prompt
2. 搜索前必须获得用户确认，避免方向偏差
3. 多轮搜索，根据发现迭代搜索策略
4. 每个论点都要有来源，区分事实与观点
5. 报告结构服务于读者理解，不是信息堆砌
6. 中英文搜索互补，扩大信息覆盖面
7. 关键事实至少2个独立来源交叉验证
