# 阶段3：确认与调整

## 目标
将生成的研究提示词展示给用户确认，根据反馈微调后进入执行阶段。

---

## 展示提示词并确认

阶段2生成提示词后，展示给用户确认：

```markdown
以下是为你生成的研究提示词，确认后我将开始多源搜索：

---
[完整提示词内容]
---

需要调整吗？如果没问题，我直接开始搜索。
```

### 用户确认规则

- 用户回复"可以"、"好"、"没问题"、"开始"、"直接搜"等 → 进入阶段4
- 用户提出具体修改意见 → 根据反馈调整提示词，再次展示确认
- 最多迭代2轮调整，之后直接进入阶段4

确认后进入阶段4：执行研究（`04-execute-research.md`）。

---

## 可选调整项

如果用户要求调整，支持以下维度：

### 调整1：详细程度

```
这个提示词的详细程度是否符合你的需求？

A. 快速版 - 简化框架，快速获取核心信息
B. 标准版 - 当前版本（推荐）
C. 完整版 - 更深入的框架和分析要求

[当前：标准版]
```

### 调整2：维度调整

```
是否需要调整优化维度？

你可以：
- 移除某个维度（如不需要输出格式要求）
- 增强某个维度（如需要更详细的子问题）
- 保持当前配置

[当前包含：研究框架、子问题拆解、信息来源指引、输出格式]
```

### 调整3：输出格式调整

```
是否需要调整输出格式？

当前格式：[摘要 + 详细分析 + 数据 + 来源]

可选调整：
- 更简洁（仅摘要+关键点）
- 更详细（增加对比表格、图表说明）
- 自定义格式
```

---

## 调整场景处理

### 场景1：调整为快速版
- 简化研究框架（保留核心结构）
- 减少子问题数量（2-3个）
- 简化输出格式（摘要+关键点）
- 保持信息来源指引

### 场景2：调整为完整版
- 增强研究框架（添加对比分析、风险评估等）
- 增加子问题数量（5-7个）
- 扩展输出格式（增加对比表、时间线图等）
- 增加信息来源要求

### 场景3：移除特定维度

**移除输出格式要求：**
```
# 研究框架
[保留]

# 核心问题
[保留]

# 信息来源
[保留]

# 输出要求
请自由组织内容，确保覆盖上述要点即可。
```

---

## 完成调整后

以清晰格式展示最终版本，然后进入阶段4执行研究。
