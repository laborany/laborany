# 阶段2：生成研究提示词

## 目标
基于阶段1的分析结果，生成包含4个优化维度的完整研究提示词。

## 优化维度

### 维度1：研究框架/结构
根据研究类型选择合适的框架（详见 `templates/research-framework.md`）：

#### 事实收集型 → 时间线+主题分类框架
```
请按照以下框架进行研究：

1. 时间线梳理
   - 按时间顺序整理事件/发言
   - 标注关键时间节点

2. 主题分类
   - 将内容按主题归类
   - 每个主题下汇总相关要点

3. 关键信息提取
   - 核心观点/事实
   - 重要数据/引用
```

#### 趋势分析型 → PEST分析框架
```
请按照PEST框架进行分析：

1. 政治环境 (Political)
   - 政策变化
   - 监管动态
   - 政府支持/限制

2. 经济环境 (Economic)
   - 市场规模
   - 投资趋势
   - 经济影响

3. 社会环境 (Social)
   - 用户接受度
   - 社会影响
   - 文化因素

4. 技术环境 (Technological)
   - 技术突破
   - 创新方向
   - 技术挑战
```

#### 观点整理型 → 多维度对比框架
```
请按照以下框架整理观点：

1. 观点分类
   - 支持方观点
   - 反对方观点
   - 中立/审慎观点

2. 代表人物/机构
   - 列出各观点的代表人物或机构
   - 简要介绍其背景

3. 论证逻辑
   - 各观点的核心论据
   - 支撑数据或案例

4. 争议焦点
   - 主要分歧点
   - 共识领域
```

#### 深度研究型 → 多维度综合框架
```
请按照以下框架进行深度研究：

1. 背景概述
   - 发展历史
   - 当前状态
   - 重要里程碑

2. 核心要素分析
   - [根据主题确定具体维度]
   - [每个维度深入分析]

3. 影响因素
   - 内部因素
   - 外部环境
   - 关键驱动力

4. 挑战与机遇
   - 主要挑战
   - 潜在机遇
   - 风险评估

5. 未来展望
   - 发展趋势
   - 预期变化
   - 不确定性
```

#### 探索了解型 → 概览+重点深入框架
```
请按照以下框架进行探索：

1. 总体概览
   - 基本概念
   - 发展现状
   - 主要方向

2. 重点领域
   - 选取3-5个重点领域深入
   - 每个领域的最新进展
   - 关键突破/事件

3. 重要人物/机构
   - 领先者名单
   - 主要贡献

4. 延伸了解
   - 推荐深入阅读的方向
```

### 维度2：具体子问题拆解

根据研究主题，拆解3-5个核心子问题：

**子问题设计原则：**
- 每个子问题应该独立且具体
- 子问题之间应该相互补充
- 覆盖研究主题的主要方面
- 避免过于宽泛或重复

**示例：**
```
核心子问题：
1. [具体问题1]：[描述要研究什么]
2. [具体问题2]：[描述要研究什么]
3. [具体问题3]：[描述要研究什么]
4. [具体问题4]：[描述要研究什么]
5. [具体问题5]：[描述要研究什么]
```

### 维度3：信息来源指引

明确应该关注的信息源类型：

**通用信息源：**
- 官方发布：官方网站、公告、白皮书
- 新闻报道：权威媒体、行业媒体
- 分析评论：专家观点、研究机构报告
- 社交媒体：官方账号、行业领袖动态
- 学术研究：论文、期刊、会议

**根据主题定制信息源：**
```
优先信息源：
1. [具体信息源类型1]：[说明为什么重要]
2. [具体信息源类型2]：[说明为什么重要]
3. [具体信息源类型3]：[说明为什么重要]

信息来源质量要求：
- 优先权威来源
- 注重时效性
- 多源交叉验证
- 标注信息发布时间
```

### 维度4：输出格式要求

规定最终输出的结构（详见 `templates/output-format.md`）：

**标准输出格式：**
```
# [研究主题] 深度研究报告

## 1. 执行摘要
[用2-3段概括核心发现]

## 2. 研究框架说明
[说明使用的研究框架]

## 3. 详细分析
[按框架分章节详细呈现]

## 4. 关键数据/引用
[列出重要数据和直接引用]

## 5. 信息来源
[列出主要信息来源]

## 6. 延伸阅读建议
[推荐进一步了解的方向]
```

## 完整提示词组装模板

将以上4个维度组装成完整提示词（详见 `templates/prompt-template.md`）：

```
# [研究主题]

## 研究目标
[清晰描述研究目标]

## 研究框架
[插入选定框架]

## 核心子问题
[插入子问题列表]

## 信息来源指引
[插入信息源指引]

## 输出要求
- 按照上述框架组织内容
- 确保信息的准确性和时效性
- 提供具体的数据和引用
- 标注信息来源和发布时间
- 使用清晰的结构和格式

## 输出格式
[插入输出格式模板]

---
请开始深入研究，尽可能全面、准确地回答上述问题。
```

## 生成流程

1. 根据研究类型选择框架模板
2. 拆解3-5个核心子问题
3. 添加信息来源指引
4. 选择输出格式
5. 组装完整提示词
6. 展示给用户确认
