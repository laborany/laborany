# 阶段1：分析请求

## 目标
深入理解用户的研究意图和需求深度，为后续优化奠定基础。

## 分析步骤

### 1. 识别核心研究主题
- 从用户请求中提取核心关键词
- 理解用户真正想要了解的内容

### 2. 判断研究类型
判断用户的请求属于以下哪种类型：

**事实收集型**
- 例："整理xxx在2026达沃斯论坛的发言"
- 特点：有明确的对象、时间、事件
- 策略：时间线+主题分类框架

**趋势分析型**
- 例："分析AI行业2026发展趋势"
- 特点：关注未来走向、行业动向
- 策略：PEST/SWOT分析框架

**观点整理型**
- 例："梳理专家对量子计算的观点"
- 特点：收集多方观点和立场
- 策略：多维度对比框架

**深度研究型**
- 例："全面研究某公司的商业模式"
- 特点：需要深入、多角度分析
- 策略：多维度综合分析框架

**探索了解型**
- 例："了解一下最新的量子计算进展"
- 特点：用户没有明确方向，需要探索
- 策略：概览+重点深入框架

### 3. 识别关键实体
提取请求中的关键信息要素：

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| 人物 | 涉及的具体人物 | xxx、专家姓名 |
| 事件 | 具体事件或活动 | 2026达沃斯论坛 |
| 组织 | 公司、机构等 | 某某公司 |
| 时间 | 时间范围 | 2026年、最近、最新 |
| 地点 | 地理位置或场景 | 达沃斯、中国 |
| 主题 | 研究的核心主题 | AI、量子计算 |

### 4. 识别研究领域

判断用户的研究主题属于哪个领域，这将决定后续搜索的权威信息源。

**领域分类（非穷举，根据实际主题灵活判断）：**

| 领域 | 典型关键词 |
|------|-----------|
| 科技/AI | 人工智能、编程、开源、模型、算法 |
| 金融/投资 | 股票、基金、加密货币、宏观经济、财报 |
| 医疗/健康 | 疾病、药物、临床试验、营养、心理健康 |
| 法律/政策 | 法规、判例、合规、知识产权、隐私 |
| 教育/学术 | 课程、论文、学校、考试、教学法 |
| 商业/管理 | 战略、营销、供应链、组织管理、创业 |
| 文化/艺术 | 电影、音乐、文学、设计、历史 |
| 科学/工程 | 物理、化学、材料、能源、航天 |
| 社会/政治 | 国际关系、社会学、人口、城市规划 |
| 生活/消费 | 美食、旅行、家居、时尚、汽车 |
| 其他 | 无法归入以上类别的主题 |

> 一个研究主题可能跨多个领域，取最主要的1-2个领域即可。

### 5. 发现领域权威信息源

**这是关键步骤。** 基于识别出的研究领域，使用 `mcp__web-search-prime__webSearchPrime` 搜索该领域的权威中英文网站和社区。

#### 搜索策略

针对识别出的领域，执行以下搜索（并行执行）：

```
搜索1（中文）："{领域名称} 权威网站 论坛 社区 推荐"
搜索2（英文）："best {domain} websites forums communities for research"
```

例如：
- 医疗领域 → `"医疗健康 权威网站 论坛 社区 推荐"` + `"best medical health websites forums communities for research"`
- 金融领域 → `"金融投资 权威网站 论坛 社区 推荐"` + `"best finance investment websites forums communities for research"`

#### 从搜索结果中提取权威源

从搜索结果中筛选出该领域最权威的网站，按以下类别整理：

```
【领域权威信息源】

领域：{识别出的领域}

中文平台（5-8个）：
| 平台 | 域名 | 适合内容 |
|------|------|---------|
| {平台名} | {domain.com} | {该平台擅长的内容类型} |
| ... | ... | ... |

英文平台（5-8个）：
| 平台 | 域名 | 适合内容 |
|------|------|---------|
| {平台名} | {domain.com} | {该平台擅长的内容类型} |
| ... | ... | ... |

通用权威来源：
- 官方网站/文档
- 权威媒体（列出该领域的权威媒体）
- 学术/报告来源（列出该领域的学术平台）
```

#### 兜底机制

如果搜索未能找到足够的领域权威源，使用以下通用平台作为兜底：

**中文通用平台：** 知乎、微博、百度学术、微信公众号搜索
**英文通用平台：** Reddit、Twitter/X、Google Scholar、Wikipedia

> 注意：这些通用平台仅作为兜底，优先使用搜索发现的领域专业平台。

### 6. 评估研究复杂度

**简单** (Level 1)
- 单一事实来源
- 明确的答案
- 例：某次会议发言的时间

**中等** (Level 2)
- 需要整合多个来源
- 需要一定分析
- 例：某人物的主要观点梳理

**复杂** (Level 3)
- 需要深入分析
- 涉及多个维度
- 例：行业趋势分析

```
关键实体：
- 人物：[列表]
- 事件：[列表]
- 时间：[范围]
- 其他：[...]

研究复杂度：[简单/中等/复杂]

【领域权威信息源】

中文平台：
- {平台1}（{domain1}）— {适合内容}
- {平台2}（{domain2}）— {适合内容}
- ...

- {平台1}（{domain1}）— {适合内容}
- {平台2}（{domain2}）— {适合内容}
- ...

```
