# 阶段1：分析请求

## 目标
深入理解用户的研究意图和需求深度，为后续优化奠定基础。

## 分析步骤

### 1. 识别核心研究主题
- 从用户请求中提取核心关键词
- 理解用户真正想要了解的内容

### 2. 判断研究类型
判断用户的请求属于以下哪种类型：

**事实收集型**
- 例："整理xxx在2026达沃斯论坛的发言"
- 特点：有明确的对象、时间、事件
- 策略：时间线+主题分类框架

**趋势分析型**
- 例："分析AI行业2026发展趋势"
- 特点：关注未来走向、行业动向
- 策略：PEST/SWOT分析框架

**观点整理型**
- 例："梳理专家对量子计算的观点"
- 特点：收集多方观点和立场
- 策略：多维度对比框架

**深度研究型**
- 例："全面研究某公司的商业模式"
- 特点：需要深入、多角度分析
- 策略：多维度综合分析框架

**探索了解型**
- 例："了解一下最新的量子计算进展"
- 特点：用户没有明确方向，需要探索
- 策略：概览+重点深入框架

### 3. 识别关键实体
提取请求中的关键信息要素：

| 实体类型 | 说明 | 示例 |
|---------|------|------|
| 人物 | 涉及的具体人物 | xxx、专家姓名 |
| 事件 | 具体事件或活动 | 2026达沃斯论坛 |
| 组织 | 公司、机构等 | 某某公司 |
| 时间 | 时间范围 | 2026年、最近、最新 |
| 地点 | 地理位置或场景 | 达沃斯、中国 |
| 主题 | 研究的核心主题 | AI、量子计算 |

### 4. 评估研究复杂度

**简单** (Level 1)
- 单一事实来源
- 明确的答案
- 例：某次会议发言的时间

**中等** (Level 2)
- 需要整合多个来源
- 需要一定分析
- 例：某人物的主要观点梳理

**复杂** (Level 3)
- 需要深入分析
- 涉及多个维度
- 例：行业趋势分析

## 输出格式

完成分析后，向用户展示分析摘要：

```
【请求分析】

核心主题：[提取的核心主题]
研究类型：[事实收集/趋势分析/观点整理/深度研究/探索了解]

关键实体：
- 人物：[列表]
- 事件：[列表]
- 时间：[范围]
- 其他：[...]

研究复杂度：[简单/中等/复杂]

---
基于以上分析，我将为您生成研究提示词...
```
