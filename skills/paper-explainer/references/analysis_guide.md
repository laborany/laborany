# 学术论文分析指南

本指南提供论文分析的结构化方法论，确保输出质量一致。

## 一、信息提取清单

### 1.1 基础元数据

| 字段 | 提取位置 | 注意事项 |
|------|----------|----------|
| 标题 | 首页顶部 | 注意副标题，用冒号分隔 |
| 作者 | 标题下方 | 保留原始顺序，通讯作者标注 * |
| 机构 | 作者下方/脚注 | 多机构用数字上标对应 |
| 发表信息 | 页眉/页脚 | 会议/期刊名、年份、DOI |

### 1.2 核心内容定位

```
Abstract     → 快速概览，提取核心贡献声明
Introduction → 研究动机、问题定义、贡献列表
Related Work → 技术背景、与现有方法的区别
Method       → 核心算法、模型架构、数学推导
Experiments  → 数据集、基线对比、消融实验
Conclusion   → 总结、局限性、未来工作
```

## 二、动机分析框架

### 2.1 三问法

1. **What**: 这篇论文解决什么问题？
2. **Why**: 为什么这个问题重要/困难？
3. **Gap**: 现有方法的不足是什么？


## 三、方法讲解策略

### 3.1 分层讲解

```
Level 1: 直觉 (Intuition)
    ↓
Level 2: 架构 (Architecture)
    ↓
Level 3: 细节 (Details)
    ↓
Level 4: 数学 (Mathematics)
```

### 3.2 公式讲解规范

对于每个关键公式：

1. **符号表**: 列出所有变量含义
2. **直觉解释**: 用自然语言描述公式在做什么
3. **维度分析**: 输入输出的shape
4. **特殊情况**: 边界条件的行为

示例：
```markdown
**注意力机制**

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

| 符号 | 含义 | 维度 |
|------|------|------|
| $Q$ | Query矩阵 | $(n, d_k)$ |
| $K$ | Key矩阵 | $(m, d_k)$ |
| $V$ | Value矩阵 | $(m, d_v)$ |
| $d_k$ | Key维度 | 标量 |

**直觉**: Query与所有Key计算相似度，用softmax归一化后加权求和Value。
**缩放因子**: $\sqrt{d_k}$ 防止点积过大导致softmax梯度消失。
```

## 四、实验分析要点

### 4.1 必读表格

- **主实验表**: 与SOTA对比
- **消融实验**: 各组件贡献
- **效率对比**: 参数量、FLOPs、推理速度

### 4.2 批判性视角

检查以下潜在问题：
- [ ] 基线是否公平（相同训练数据/超参）
- [ ] 数据集是否有代表性
- [ ] 指标是否全面（不只看准确率）
- [ ] 是否有统计显著性分析
- [ ] 消融实验是否充分

## 五、图表处理

### 5.1 图片分类

| 类型 | 特征 | 处理方式 |
|------|------|----------|
| 框架图 | 展示整体架构，含模块框和箭头，通常宽>高 | **嵌入method开头**，详细描述数据流 |
| 模块细节图 | 单个组件内部结构 | 嵌入method对应段落 |
| 实验曲线 | 折线图/柱状图，含坐标轴 | **嵌入experiments**，解释趋势 |
| 可视化结果 | 热力图/注意力图/生成样本 | 嵌入experiments定性分析处 |
| 示意图 | 概念解释/问题对比 | 嵌入motivation帮助理解 |
| 其他 | Logo/装饰/无关 | 仅放附录或忽略 |

### 5.2 图片分类方法

**Step 1: 尺寸分析**
- 宽度 > 600px 且 宽 > 高: 可能是框架图
- 宽高接近且较小: 可能是实验结果图或图标

**Step 2: 页码定位**
- 第1-2页: 多为概念示意图或问题说明
- Method章节页: 多为架构图、模块图
- Experiments章节页: 多为实验曲线、可视化

**Step 3: 正文引用匹配**
- 搜索 "Figure 1"、"Fig. 2" 等引用
- 根据引用上下文判断图片用途
- 提取论文中的图注(caption)作为说明

**Step 4: 内容特征**
- 含箭头、模块框: 架构图
- 含坐标轴、图例: 实验图
- 含热力色彩: 可视化结果

### 5.3 嵌入策略

**框架图嵌入** (最重要):
```html
<figure class="embedded-figure architecture">
  <img src="..." alt="模型架构">
  <figcaption>
    <strong>图1: 模型整体架构</strong><br>
    本文提出的模型由三个核心模块组成：(1) 编码器负责...
    (2) 注意力模块实现... (3) 解码器生成...
  </figcaption>
</figure>
```

**实验图嵌入**:
```html
<figure class="embedded-figure experiment">
  <img src="..." alt="实验结果">
  <figcaption>
    <strong>图3: 不同方法在数据集X上的对比</strong><br>
    从图中可以观察到：(1) 本文方法在所有指标上优于基线...
    (2) 随着数据量增加，优势更加明显...
  </figcaption>
</figure>
```

### 5.4 图例格式

```markdown
**图 X**: [标题]
- **类型**: [框架图/实验图/示意图/可视化]
- **内容**: [图中展示了什么]
- **关键观察**: [最重要的发现]
- **嵌入建议**: [应嵌入到哪个章节]
- **与正文关联**: [对应论文哪个论点]
```
